What is Zoo on Web?

  1. Anca Dorneanu
  2. Dan Dragomirescu
  3. Rares Munteanu

Abstract

Proiectul "Zoo on Web" reprezinta o aplicatie web ce gestioneaza informatiile ce vizeaza animalele gazduite intr-o gradina zoologica. Se redau, intr-o maniera facila si accesibila, informatii legate de animale, precum: denumire stiintifica, denumire populara, specie, familia din care face parte, origine, habitat, mod de alimentatie, descriere, statut, proprietati specifice, etc.

De asemenea, acest proiect are in vedere mecanisme de cautare/filtrare/afisare multi-criteriala si importarea/exportarea datelor in format JSON si XML.

Introducere

Proiectul propune o solutie pentru gestionarea eficienta si rapida (prin prisma tehnologiilor folosite) a informatiilor despre animalele dintr-o gradina zoologica. Accentul se pune pe cautarea/filtrarea multi-criteriala.


Etapele dezvoltarii proiectului

  1. Crearea arhitecturii aplicatiei
  2. Modelarea bazei de date
  3. Front-end: crearea paginilor HTML si stilizarea acestora utilizand CSS
  4. Back-end: design-ul si crearea claselor PHP
  5. Utilizarea API-urilor: PayPal
  6. Adaugarea functionalitatilor utilizand JavaScript si AJAX
  7. Testare

Tehnologii utilizate

Front-end

Pe partea de front-end, tehnologiile folosite sunt: HTML5, CSS3, JavaScript. Tot la acest capitol, se va furniza si un search autocomplete folosind tehnologia AJAX.

Back-end

Pe partea de back-end, am utilizat PHP.

De asemenea, pe partea ce tine de baze de date, am utilizat MySQL, impreuna cu PHP.

API: Application Program Interface

In ceea ce priveste API-urile, vom utiliza PayPal API.

Arhitectura aplicatiei

Home

Pe pagina principala (home page), avem un navbar cu butoane spre alte pagini: "login", "categories", "contact", astfel incat fiecare buton directioneaza catre o pagina separata ce gestioneaza o anumita functionalitate.

Search

Search-ul ofera functionalitatea de a cauta un animal intr-un mod usor (dupa nume) sau in modul avansat, dupa mai multe criterii.

Categories

Pe aceasta pagina se vor gasi categoriile de animale, dupa cum urmeaza: mamifere, reptile si pasari.

Tot pe aceasta pagina, se pot importa date in format XML sau JSON ce vor fi inserate in baza de date sub format MySQL

Contact

Pagina de contact, unde se vor gasi informatii precum adresa, numar de telefon, email, etc. De asemenea, tot in cadrul acestei pagini, se poate face o donatie ce integreaza un API REST de la PayPal. Si tot aici se va gasi un formular pentru adaugarea de sugestii.

Login

Login-ul ofera functionalitatea de a te autentifica drept administrator (ce poate adauga/sterge un animal sau o categorie, modifica anumite date legate de un animal/categorie) sau ca donator (in caz ca cineva doreste sa faca o donatie financiara via PayPal).

Stocarea datelor: MySQL

Orice user si orice animal, respectiv orice detaliu despre acestea, vor fi stocate in tabele separate. De asemenea, se mai stocheaza intr-un alt tabel si sugestiile provenite de la useri.

Tabelele bazei de date

Animals

Tabela contine 20 de campuri, dupa cum urmeaza: ID, link (picture path), popular_name, scientific_name, category, family, description, origin, status, climate, habitate, dangerousness, special_prop, related_species, enemies, nourishment, breeding, fur, fur_color, training. Aceasta tabela este indexata pe campul ID.

Users

Tabela contine 8 campuri: id, name, username, password, email, phone, city, birthdate.

Suggestions

Tabela contine 4 campuri: id, name, email, subject.


Este de specificat ca nu exista corelari intre tabelele din baza de date, intrucat nu a fost nevoie.
