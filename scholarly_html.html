Abstract

Proiectul "Zoo on Web" reprezinta o aplicatie web ce gestioneaza informatiile ce vizeaza animalele gazduite intr-o gradina zoologica. Se redau, intr-o maniera facila si accesibila, informatii legate de animale, precum: denumire stiintifica, denumire populara, specie, familia din care face parte, origine, habitat, mod de alimentatie, descriere, statut, proprietati specifice, etc.

De asemenea, acest proiect are in vedere mecanisme de cautare/filtrare/afisare multi-criteriala si importarea/exportarea datelor in format JSON si XML.

Introducere

Proiectul propune o solutie pentru gestionarea eficienta si rapida (prin prisma tehnologiilor folosite) a informatiilor despre animalele dintr-o gradina zoologica. Accentul se pune pe cautarea/filtrarea multi-criteriala.

Etapele dezvoltarii proiectului

  1. Crearea arhitecturii aplicatiei
  2. Modelarea bazei de date
  3. Front-end: crearea paginilor HTML si stilizarea acestora utilizand CSS
  4. Back-end: design-ul si crearea claselor PHP
  5. Utilizarea API-urilor: Google Calendar & PayPal
  6. Adaugarea functionalitatilor utilizand JavaScript si AJAX
  7. Testare

Tehnologii utilizate

Front-end

Pe partea de front-end, tehnologiile folosite sunt: HTML5, CSS3, JavaScript. Tot la acest capitol, se va furniza si un search autocomplete folosind tehnologia AJAX.

Back-end

Pe partea de back-end, vom utiliza PHP.

De asemenea, avand in vedere ca se pune accentul pe cautare (rapiditate si eficienta), vom folosi Redis ("in-memory data structure store, used as database"). Pentru ca acesta se bazeaza pe date de tipul cheie-valoare, deci NU este relational (NoSQL), acesta functioneaza intr-un mod foarte rapid.

De mentionat este ca, in cazul in care vom intampina probleme in realizarea back-end-ului in aceasta maniera, vom apela la MySQL in loc de Redis.

API: Application Program Interface

In ceea ce priveste API-urile, vom utiliza Google Calendar API si PayPal API.

Arhitectura aplicatiei

Pe pagina principala (home page), avem un navbar cu butoane spre alte pagini: "login/sign up", "categories", "calendar", "contact", "buy tickets", astfel incat fiecare buton directioneaza catre o pagina separata ce gestioneaza o anumita functionalitate.

Login/Sign up

Login-ul ofera functionalitatea de a te autentifica drept administrator (ce poate adauga/sterge un animal sau o categorie, modifica anumite date legate de un animal/categorie) sau ca donator (in caz ca cineva doreste sa faca o donatie financiara via PayPal).

Tot pe aceasta pagina se va afla si un formular de sign up pentru donatori.

Categories

Pe aceasta pagina se vor gasi categoriile de animale, in functie de niste criterii care mai raman de stabilit (in principiu, vor fi criteriile cerute in cazul cautarii).

Calendar

Folosind Google Calendar API, un vizitator poate vedea anumite evenimente ce au loc in cadrul gradinii zoolgice (de exemplu dresarea unui leu intr-o anumita zi).

Buy tickets

O pagina care ofera functionalitatea de a cumpara un bilet. Biletul este achitat prin intermediul PayPal API.

Contact

Pagina de contact, unde se vor gasi informatii precum adresa, numar de telefon, email, etc.

Stocarea datelor: Redis

Orice user si orice animal, respectiv orice detaliu despre acestea, vor fi stocate sub forma de obiecte separate. Din moment ce nu exista tabele, nu vor exista generalizari/abstractizari asupra animalelor.

Servicii

Github

GitHub este un serviciu de gazduire web pentru proiecte de dezvoltare a software-ului care utilizeaza sistemul de control al versiunilor Git. GitHub ofera planuri tarifare pentru depozite private, si conturi gratuite pentru proiecte open source.

Scholarly HTML

Scholarly HTML este un format de date 'domain-specific', ce utilizeaza Scholarly HTML, si e bazat in totalitate pe 'open standards', ce permite schimbul de articole stiintifice intr-un mod care este compatibil cu browsere 'off-the-shelf'.
